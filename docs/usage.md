# 使用说明

## 界面介绍

打开浏览器访问服务器地址（默认 `http://localhost:8080`），即可看到测速界面。

### 主要区域

1. **速度显示区**：展示下载速度和上传速度，带实时进度条,同时读出延迟 / 抖动 / Bufferbloat 分级
2. **指标网格**：显示丢包率、Bufferbloat、IP 地址和连接方式(IPv4 / IPv6)
3. **开始按钮**：点击开始测速
4. **历史记录**(需服务端开 `SPEEDTEST_DB_PATH`)：分页表格 + 一键导出 CSV/JSON
5. **页脚版本号**：显示当前跑的 `vX.Y.Z`(悬停可看 commit + 构建日期);本地
   `go run` 时显示 `dev`

### 历史记录列

- **时间** / **来源 IP** / (可选)**归属地** / ↓Mbps / ↑Mbps / 延迟 /
  Bufferbloat
- "来源 IP"是服务端 `ClientIP()` 提取的真实对端(支持可信反向代理白名单)
- "归属地"只有在服务端启用 IP 地理定位 (`SPEEDTEST_GEOIP_DB` 或
  `SPEEDTEST_GEOIP_LICENSE_KEY`) 时才显示;未启用则整列隐藏
- 旧记录若在归属地开启前写入,该列显示 `--`

### 控制按钮

- **语言切换**(右上角):在中英文之间切换
- **主题切换**(右上角):在暗色/亮色主题之间切换，默认跟随系统偏好

## 测速流程

点击"开始测速"按钮后，系统会依次执行：

1. **延迟测量**：发送 20 次 Ping 请求，计算平均延迟、抖动和丢包率
2. **下载测速**：通过多个并行连接下载数据，实时显示下载速度
3. **上传测速**：通过多个并行连接上传数据，实时显示上传速度

测速完成后，按钮变为"重新测速"，可再次点击进行测试。

## 测速模式

服务器支持两种测速模式，通过 `SPEEDTEST_MODE` 环境变量配置：

### 时间模式（time）

默认模式。在固定时间内持续传输数据，测量平均速度。

- 持续时间由 `SPEEDTEST_DURATION` 控制（默认 15 秒）
- 适合测量稳定的带宽速度

### 数据量模式（size）

传输固定大小的数据，测量完成速度。

- 下载大小由 `SPEEDTEST_DOWNLOAD_SIZE` 控制（默认 25 MB）
- 上传大小由 `SPEEDTEST_UPLOAD_SIZE` 控制（默认 10 MB）
- 适合快速测试或带宽较小的环境

## 注意事项

1. **首次测速**：由于 TCP 慢启动机制，首次测速结果可能略低于实际带宽。系统会在计算时跳过前 2 秒的预热阶段。

2. **并行连接**：默认使用 4 个并行连接，可更准确地测量高带宽链路的实际速度。
   下拉框可选 1/2/4/8/16,但**不能超过服务端 `SPEEDTEST_MAX_CONCURRENT`**
   (默认 10);超过的选项前端会自动禁用并标注 `(>N)`,避免必然 503 的组合。

3. **浏览器兼容性**：支持所有现代浏览器（Chrome、Firefox、Safari、Edge），需要支持 `fetch` API 和 `ReadableStream`。

4. **代理环境**：如果服务器部署在反向代理（如 Nginx）后面，确保正确配置 `X-Forwarded-For` 或 `X-Real-Ip` 头，以便正确显示客户端 IP。

## 常见问题

### 测速结果不准确

- 确保测试期间没有其他大流量应用占用带宽
- 尝试增加 `SPEEDTEST_DURATION` 或 `SPEEDTEST_DOWNLOAD_SIZE` 获得更稳定的结果
- 检查是否有浏览器插件干扰网络请求

### 页面无法加载

- 检查服务器是否正常运行：`curl http://localhost:8080/healthz | jq`(顺便看
  version / uptime / accepted-vs-rejected 计数)
- 检查防火墙是否放行了对应端口
- 确认绑定的地址和端口配置正确

### 上传测速失败

- 某些反向代理可能有请求体大小限制，需要调整配置
- 检查浏览器控制台是否有 CORS 或网络错误

### 测速时出现"系统繁忙 (503)"

- 说明并发达到 `SPEEDTEST_MAX_CONCURRENT` 上限。要么调低前端选的 streams 数,
  要么运维侧调高上限
- 前端会自动尝试 Retry;点击 toast 上的"重试"即可

### 版本号不更新 / PWA 缓存陈旧

- PWA 的 shell 缓存 key 会随每次 release 自动变(`speedtest-<version>`),下
  次访问自动新装
- 强制刷新一次(Ctrl+Shift+R)可绕过所有缓存
- 页脚上悬停"vX.Y.Z"能看到实际跑的 commit 短 hash + 构建日期
